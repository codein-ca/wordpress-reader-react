The idea here is to cache images for offline use.

  1. Upon loading - connect to the API and load 3 images async.
  2. If network is not available, just continue serving the assets previously acquired (looping the current available set).
  3. If network is available, retrieve a new set of 3 images (anb obviously these should be the new fresh cache).

In order to validate the service worker behavior (regarding caching) all images should have always the same name (a, b and c).
